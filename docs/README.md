以下の順序で読むことを推奨します。

| No. | ファイル | 内容 |

## 読む順番

1. **01_requirements.md** — まずプロジェクトの全体像を理解する
2. **02_design.md** — 設計方針とディレクトリ構成を理解する
3. **03_package_api.md** — 公開 API の仕様を確認する
4. **04_implementation_plan.md** — 実装の優先順位とタスクを確認する
5. **07_test_plan.md** — テスト方法を確認する
6. **08_documentation_spec.md** — ドキュメントの構成を確認する
